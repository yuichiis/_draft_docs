NDArray インターフェイス
-----------------
### 概要
Rindow 数学行列は、同じタイプの「アイテム」のコレクションを記述する N 次元配列タイプである NDArray を提供します。 項目には、たとえば N個の float 値を使用してインデックスを付けることができます。

NDArray に含まれるすべての項目は同じ型値を持ちます。 すべての項目はPHPの標準インターフェースである「ArrayAccess」で指定されます。 1 つの値のデータ型は、NDArray で定義された整数型、浮動小数点型、またはブール型です。

N 次元配列は 1 次元配列バッファにマッピングされ、連続領域として格納されます。

![NDArray](images/ndarray.png)

最も重要なことは、NDArray インターフェイスは Rindow フレームワークの一部ではないということです。 **インターフェイスは独立して定義されています**。他のフレームワークは自由に実装できます。

```php
use Interop\Polite\Math\Matrix\NDArray;
```

[interop-phpobjects/polite-math](https://github.com/interop-phpobjects/polite-math) を参照してください。


バッファオブジェクト
-------------
＃＃＃ 概要
バッファ オブジェクトは、NDArray の実際のデータを格納する領域です。 1次元配列を実装します。

PHP の標準 ArrayAccess インターフェイスと Countable インターフェイスを実装する必要があります。
様々な実装をすることを前提としているため、基本のBufferのインターフェースが定義されています。

1 次元配列はどのような方法でも実装できますが、一般にメモリ領域が連続していると、CPU による高速演算の実行が容易になります。 C言語レベルでメモリ領域の参照が容易で、高速演算ライブラリー間のデータのやり取りも容易です。

これらの理由から、NDArray は PHP 配列ではなく 1 次元配列バッファ オブジェクトを使用します。
